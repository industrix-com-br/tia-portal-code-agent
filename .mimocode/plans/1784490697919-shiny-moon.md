## Ações Planejadas

### 1. Criar Branches Recomendadas

Criar a estrutura de branches padrão para desenvolvimento:

```
main (protegido)
├── develop (branch de desenvolvimento principal)
├── release/v0.0.1 (preparação de release)
├── feature/* (branches de funcionalidade)
└── hotfix/* (correções urgentes)
```

**Branches a criar:**
- `develop` — branch de integração principal
- `release/v0.0.1` — branch de preparação para release (versão atual)

### 2. Configurar Proteção da Branch `main`

Configurar regras de proteção via API:
- ✅ Require pull request before merging
- ✅ Require approvals (1 approval mínimo)
- ✅ Require status checks to pass (quando CI existir)
- ✅ Require branches to be up to date
- ✅ Require conversation resolution
- ✅ Do not allow bypassing the above settings
- ✅ Restrict who can push to matching branches (apenas via PR)

### 3. Configurar Proteção da Branch `develop`

Configurar regras de proteção:
- ✅ Require pull request before merging
- ✅ Require approvals (1 approval)
- ✅ Allow force pushes (para integração)
- ✅ Allow deletions (para cleanup)

### 7. Criar Branch Protection Rules via API

```bash
# Proteger main
gh api -X PUT repos/allanmarum/tia-portal-code-agent/branches/main/protection \
  --input - <<EOF
{
  "required_status_checks": null,
  "enforce_admins": true,
  "required_pull_request_reviews": {
    "required_approving_review_count": 1
  },
  "restrictions": null,
  "required_linear_history": false,
  "allow_force_pushes": false,
  "allow_deletions": false,
  "block_creations": false,
  "required_conversation_resolution": true
}
EOF

# Proteger develop (mais permissiva)
gh api -X PUT repos/allanmarum/tia-portal-code-agent/branches/develop/protection \
```

### 8. Configurar Dependabot (Recomendado)

Criar `.github/dependabot.yml` para atualizações automáticas:

```yaml
version: 2
updates:
  - package-ecosystem: "nuget"
    directory: "/"
    schedule:
      interval: "weekly"
    open-pull-requests-limit: 10
```

1. Criar branch `develop`
2. Criar branch `release/v0.0.1`
3. Proteger branch `main` (API)
4. Proteger branch `develop` (API)
5. Atualizar configurações do repositório (descrição, topics, merge options)
6. Criar labels padrão
7. Criar issue templates (opcional)
8. Criar dependabot.yml

## Comandos Principais

```bash
# 1. Criar branches
git checkout -b develop
git push origin develop
git checkout main
git checkout -b release/v0.0.1
git push origin release/v0.0.1
git checkout main

# 2. Proteger branches (via API)
gh api -X PUT repos/allanmarum/tia-portal-code-agent/branches/main/protection ...
gh api -X PUT repos/allanmarum/tia-portal-code-agent/branches/develop/protection ...

# 3. Atualizar repositório
gh repo edit --description "..." --homepage "..." --enable-issues --enable-projects ...
gh repo edit --delete-branch-on-merge --allow-update-branch

# 4. Criar topics
gh repo edit --add-topic tia-portal,siemens,plc,add-in,mcp,ai-agent,dotnet,engineering

# 5. Criar labels
gh label create "bug" --color "d73a4a" --description "Something isn't working"
# ... outros labels
```
