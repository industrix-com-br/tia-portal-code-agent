## Critical SDK Finding

- `ModelContextProtocol` v1.4.1+ targets `netstandard2.0` (compatible with net48)
- `ModelContextProtocol.AspNetCore` targets `net8.0` only
- **Decision:** MCP HTTP server runs in a separate .NET 8.0 process. The Add-In manages its lifecycle. This aligns with the spec's "Architecture B" (external MCP Host) but implemented pragmatically from day 1 to avoid forced rewrites.

- Change: compile failure scenario
- Change: expired approval → APPROVAL_EXPIRED

## Key Architectural Decisions

| Decision | Choice | Rationale |
|---|---|---|
| MCP hosting | Separate net8.0 process | ASP.NET Core requires net8.0; avoids forced rewrites later |
| IPC between Add-In and MCP Host | Named Pipe (prepare) | Windows-native, ACL-protected, spec-aligned |
| Contracts TFM | netstandard2.0 | Compatible with both net48 and net8.0 consumers |
| Simulator strategy | In-memory ITiaProjectService | Enables full tool testing without TIA Portal |
| Hash algorithm | SHA-256 (canonical JSON) | Deterministic, reusable for concurrency checks |
| Selection token format | GUID + metadata | Opaque, session-scoped, expires on project close |
| Error model | Structured TiaError | Code + message + retryable + correlationId + details |
| Write approval | Preview → Diff → Token → Validate → Apply | Mandatory sequence, no shortcuts |

## Remaining Limitations (post-implementation)

These items genuinely cannot be completed without a real TIA Portal V21 installation:

1. **Real Openness adapter** (Phase 6) — requires TIA running with a project open
2. **Real selection capture** — requires Add-In loaded in TIA Portal
3. **Real compilation** — requires TIA compilation engine
4. **Real block export/import** — requires Openness export API
5. **WPF-based Add-In UI** — requires Visual Studio for designer support
6. **Publisher packaging** — requires Siemens Publisher tool
7. **OpenCode integration** — requires running OpenCode server with model provider

All of these are isolated behind `ITiaProjectService` and can be implemented independently once TIA Portal is available.
